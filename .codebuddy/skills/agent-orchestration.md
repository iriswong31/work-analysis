# 数字分身Agent协作系统

## 🚀 快速开始（推荐）

**你只需要说自然语言，协调中心会自动编排所有Agent：**

| 你说的话 | 系统自动执行 |
|----------|--------------|
| "开始今天的工作" | 规划Agent制定计划 → 等你确认 |
| "好的，开始执行" | 执行Agent执行 → 检查Agent验收 → 等你反馈 |
| "这个需要改xxx" | 检查Agent解析 → 执行Agent修改 → 重新验收 |
| "通过，下一个" | 标记完成 → 执行下一个任务 |
| "今天结束了" | 复盘Agent总结 → 基于你的反馈更新记忆 |

## 每日工作流程

### 1. 早晨启动（规划Agent）
```
触发条件：新的一天开始 / Iris说"开始今天的工作"

执行步骤：
1. 读取记忆文件（L0-L4 + Intent）
2. 检查昨日复盘Agent的记忆更新
3. 根据目标和状态制定今日规划
4. 输出任务列表给执行Agent
```

### 2. 日间执行（执行Agent + 检查Agent）
```
循环执行：

执行Agent:
1. 按优先级执行任务
2. 产出交付物
3. 提交给检查Agent

检查Agent:
1. 验收交付物
2. 输出验收报告给Iris
3. 等待Iris反馈
4. 将反馈转化为修改指令
5. 传递给执行Agent

迭代直到：
- 交付物通过验收
- 或当天时间结束
```

### 3. 傍晚复盘（复盘Agent）
```
触发条件：当天工作结束 / Iris说"复盘今天的工作"

执行步骤：
1. 收集当天所有信息
2. 汇总任务完成情况
3. 提取洞见
4. 分析Iris的所有反馈
5. 仅基于Iris反馈更新记忆
6. 输出明日建议给规划Agent
```

## 使用指南

### 调用规划Agent


## 记忆系统映射

```
记忆层级              │ 更新来源            │ 更新频率
─────────────────────┼────────────────────┼──────────
L0_状态层             │ 自动 + Iris反馈     │ 实时
L1_情境层             │ 自动 + Iris反馈     │ 每日
L2_行为层             │ 仅Iris反馈         │ 按需
L3_认知层             │ 仅Iris反馈         │ 月度
L4_核心层             │ 仅人工修改         │ 年度
```

## 关键原则

### 1. Iris反馈是最高优先级
所有Agent都必须尊重Iris的反馈，反馈内容直接影响执行方向。

### 2. 记忆更新严格来源限制
复盘Agent更新记忆时，必须标注来源的Iris原话，不能自行推断。

### 3. 以终为始
规划Agent制定计划时，始终从长期目标倒推，确保每日工作服务于大目标。

### 4. 持续迭代
检查Agent和执行Agent的循环迭代，直到产出达到Iris满意的标准。
