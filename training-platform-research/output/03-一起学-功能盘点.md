# 一起学 - 产品能力盘点

腾讯基金会先后做了两个版本的"一起学"平台：

| 版本 | 搭建方式 | 小程序主体 | 定位 | 当前状态 |
|-----|---------|-----------|------|---------|
| **乐享版** | 腾讯乐享平台搭建 | 深圳市腾讯计算机系统有限公司 | 线上课程学习 | 并行使用 |
| **自研版** | 腾讯基金会自研 | 深圳市腾讯计算机系统有限公司 | 机构管理、线下培训活动管理 | 并行使用 |

**两个版本并行的原因**：
- 最开始使用乐享搭建学习平台，但乐享平台在个性化培训过程无法满足需求，且无法生成独立的小程序
- 因此自研开发了新版一起学平台
- 由于两个平台各有优势，在一段时间中两个平台并行使用
- **乐享版**：主要用于线上学习
- **自研版**：用于机构管理、线下培训活动管理

---

## 一、自研版 - 产品能力盘点

### 1.1 平台定位

腾讯基金会自研的公益行业资源链接与活动管理平台，特点：

1. **"办会"而非"办班"**：核心是活动/会议的全流程管理，而非培训班管理
2. **知识沉淀导向**：办会的目的是把会议内容沉淀成平台知识，用户可翻看历届会议的所有信息材料
3. **资源链接平台**：汇聚专家库、机构库、案例库、工具包、公益报告等行业资源
4. **课程依赖乐享**：课程内容跳转乐享版，自身不承载视频课程
5. **报名依赖外链**：资助计划报名跳转微信公众号，非平台内闭环

### 1.2 平台架构

| 应用端 | 使用对象 | 主要功能 |
|--------|---------|---------|
| 管理后台 | 平台运营人员、入驻机构 | 机构管理、活动管理、内容管理、资助计划配置等 |
| C端 | 公益从业者、机构成员、专家 | 小程序"一起学"，活动报名、资源浏览、课程跳转等 |

### 1.3 管理后台

| 能力分类 | 功能板块 | 主要功能 |
|---------|---------|---------|
| 角色管理 | 机构管理 | **机构列表**：查看/添加合作机构，支持审核、封禁<br>**机构审核**：机构入驻审核 |
| 角色管理 | 专家管理 | 专家邀请、审核管理、定向邀请 |
| 办会工具 | 活动管理 | **活动类型**：单场活动、系列活动、大型会议、峰会<br>**活动创建7步流程**：选择类型→基础信息→嘉宾管理→议程管理→FAQ配置→提交上架→成果上传<br>**活动详情Tab**：基本信息、议程管理、海报与邀请函、嘉宾管理、参会管理、常见问题、成果管理、核销管理<br>**活动看板**：全部/待审核/未开始/进行中/已结束 |
| 办会工具 | 历届大会 | 历届大会归档展示 |
| 办会工具 | 其他行业会议 | 行业会议管理 |
| 内容管理 | 课程管理 | 课程列表、上架编辑，**实际跳转乐享版**，自身不承载课程视频 |
| 内容管理 | 问卷管理 | 问卷创建与发放 |
| 资源中心 | 运营管理 | **公益报告管理**：议题报告、前瞻研究、行动研究<br>**工具包管理**：公益工具资源<br>**案例库管理**：公益案例展示<br>**机构库管理**：合作机构展示<br>**专家库管理**：专家资源展示<br>**资助计划管理**：项目信息展示+外链报名（如深根者计划跳转公众号报名） |
| 平台管理 | 运营管理 | 广告管理、联系渠道、底部导航、首页机构推荐 |
| 平台管理 | 系统管理 | 用户管理、角色管理、菜单管理、字典管理 |

### 1.4 角色体系

| 角色组 | 角色 | 权限范围 |
|-------|------|---------|
| 平台运营方 | 超级管理员、运营人员 | 全权限，管理所有内容和机构 |
| 合作机构 | 机构管理员、机构成员 | 管理本机构信息，发布活动 |
| 专家 | 受邀专家 | 专家身份展示，参与活动 |
| 普通用户 | 公益从业者 | 浏览资源、报名活动、学习课程 |

### 1.5 平台局限性

| 维度 | 说明 |
|-----|------|
| 课程能力 | ❌ 依赖乐享版，自身不承载课程视频 |
| 报名能力 | ❌ 资助计划报名跳转外链（微信公众号），非平台内闭环 |
| 培训班管理 | ❌ 缺少培训班/学习任务管理能力 |
| 学习追踪 | ❌ 缺少签到打卡、学习进度追踪等培训过程管理 |

**适合场景**：公益行业会议/论坛组织、公益机构资源链接、项目信息展示与招募引流  
**不适合场景**：系统化培训班管理、学员学习过程追踪、线上线下结合的培训项目

## 二、乐享版 - 产品能力盘点

### 2.1 平台定位

腾讯乐享是面向企业的一站式智慧型学习社区平台，提供企业培训、文化建设、知识管理一体化解决方案。在一起学体系中，乐享版作为线上课程学习的承载平台。

**乐享平台特点**：
- 丰富的应用、组件化的配置、全流程的运营服务
- 萃取腾讯十余年管理经验，服务20W+企业客户
- 致力于提升企业效能，让企业与人才同生共长

### 2.2 平台架构

| 应用端 | 使用对象 | 主要功能 |
|--------|---------|---------|
| 乐享管理后台 | 腾讯基金会运营人员 | 课程管理、学员管理、项目管理、考试管理、数据统计等 |
| C端 | 公益从业者 | 通过一起学小程序跳转乐享，观看课程、参与培训项目 |

### 2.3 管理后台

| 能力分类 | 功能板块 | 主要功能 |
|---------|---------|---------|
| 课堂管理 | 课程管理 | **自建课程**：开课内容配置（主题、章节模式、素材最多300个）、课程信息（分类、介绍、标签、封面支持**智能生成图片**）<br>**权限设置**：必修/选修学员范围、未完成自动催办<br>**AI高级设置**：AI文稿辅助观看、**AI讲师答疑**（学员可通过AI对话询问课程问题）<br>**防作弊设置**：人脸验证、视频防快进、打卡提醒<br>**课程激励**：积分、学分、**自动颁发证书** |
| 课堂管理 | 素材库管理 | **采购素材库**（乐课商城免费/付费课程）+ **自建素材库** |
| 课堂管理 | 乐课商城 | **课程分类**：企业战略、领导力、职业素养、行业课程、岗位技能、党建专区<br>**采购模式**：免费采购 / 付费采购（按年计费）<br>**供应商**：腾讯乐享课程 + 外部供应商（凯洛格、馒头商学院等20+内容提供方） |

## 三、自研版与乐享版对比

### 3.1 "办班"与"办会"的区别

| 维度 | 乐享版（办班） | 自研版（办会） |
|-----|--------------|--------------|
| **目标** | 人才培养，学员要"学会"某些东西 | 信息交流+知识沉淀，参会者"参与"，后来者"查阅" |
| **周期** | 长周期（几周到几个月） | 短周期（几小时到几天） |
| **学员关系** | 持续跟踪，有学习进度、考核、证书 | 一次性参与，但会议内容长期可查 |
| **内容形式** | 课程+作业+考试+实践任务 | 议程+嘉宾演讲+互动环节 |
| **核心能力** | 学习任务编排、进度追踪、考核评估、证书发放 | 议程管理、嘉宾管理、报名核销、成果归档 |
| **过程管理** | 签到+作业+考试+打分（双向评估） | 签到核销（到场即可） |
| **产出** | 培养出合格学员（有证书/考核结果） | 沉淀的会议资产（直播录屏、资讯报道、报告、嘉宾资料） |

### 3.2 能力互补关系

| 能力维度 | 自研版 | 乐享版 | 互补方式 |
|---------|-------|-------|---------|
| **课程承载** | ❌ 不承载 | ✅ 完整 | 自研版课程跳转乐享版播放 |
| **活动管理** | ✅ 完整 | ❌ 无 | 活动/会议管理用自研版 |
| **机构管理** | ✅ 有 | ❌ 无 | 机构入驻、审核用自研版 |
| **资源中心** | ✅ 有 | ❌ 无 | 专家库、案例库等用自研版 |
| **培训班管理** | ❌ 无 | ✅ 完整 | 系统化培训用乐享版 |
| **学习追踪** | ❌ 无 | ✅ 完整 | 学习进度、考核用乐享版 |
| **数据分析** | ❌ 基础 | ✅ 企业级 | 数据分析用乐享版 |
| **AI能力** | ❌ 无 | ✅ 有 | AI出题、AI讲师答疑用乐享版 |
