## 一、项目背景

"耕耘者"振兴计划是2021年5月24日，农业农村部与腾讯公司在北京签署的战略合作协议，面向乡村治理骨干和新型农业经营主体带头人开展培训，共同推进乡村人才振兴。

**业务目标**：
- 围绕提升乡村治理骨干人才的治理能力、新型农业经营主体引领带动小农户发展能力
- 通过线上线下相结合的方式
- **3年内实现线下免费培训10万人次**

**产品目标**：
基于3年完成10万人次线下培训的业务目标，需要在"为村耕耘者知识分享平台"上开发能够满足线下培训开班全流程管理的产品。

## 二、培训业务流程体系

耕耘者线下培训梳理了 **5个培训作业流程**：

```
1.招生流程 → 2.开班流程 → 3.训后运营流程 → 4.质检流程 → 5.资源管理流程
```

| 流程 | 名称 | 优先级 | 流程范围 |
|-----|------|--------|---------|
| 1 | **招生流程** | 高 | 对外推广、与意向单位/学员对接，确认开班意愿与排期 |
| 2 | **开班流程** | 高 | 开班前中后全任务，包括安排场地、时间、讲师、差旅住宿、课堂授课、课堂纪律管理、发放学习证明等 |
| 3 | **训后运营流程** | 中 | 学员后续作业、是否落地培训内容的持续跟进与辅导工作任务 |
| 4 | **质检流程** | 中 | 周期性检查合作培训机构是否按照要求完成开班任务，包括暗访、数据监控、开班评审、跟班检查等 |
| 5 | **资源管理流程** | 高（流程1&2依赖） | 四大资源流程管理：课程、讲师、合作机构、研学基地、教学物料 |

## 三、培训对象分类

### 3.1 两大培训类型

| 培训类型 | 培训对象 | 身份细分 |
|---------|---------|---------|
| **乡村治理骨干** | 村支书等基层治理人员 | 村支书（可邀请本村一位干部参与学习） |
| **新型农业经营主体** | 农业经营带头人 | 农民专业合作社理事长、家庭农场主、辅导员 |

### 3.2 开班类型体系

**乡村治理骨干**：
- 分阶培训：第一阶段-送教 → 第二阶段-基地研学班 → 第三阶段-产业能力提升班
- 示范班
- 调训班

**新型农业经营主体**：
- 分阶培训：第一阶段-送教 → 第二阶段-基地研学班 → 第三阶段-产业能力提升班
- 示范班：标准班
- 负责人现场培训班：大豆粮油、其他
- 调训班

## 五、招生流程详解

### 5.1 流程概述

招生流程用于指导规范耕耘者线下开班招生时工作流程，提高招生效率与保障用户报名体验。

**流程分为两个阶段**：获取线索 → 线索转化

**4个流程相关人**：管理员、监督者、第三方培训机构、用户

**13个主要步骤**

### 5.2 乡村治理骨干班招生流程

**角色与职责**：

| 角色 | 主要职责 |
|-----|---------|
| 监督员（农业农村部） | 查看招生数据（注册账号人数、申请报名人数、报名成功人数、完成学习考试人数、队列人数、预期开班时间） |
| 腾讯管理员 | 发布招生信息 → 进入线下开班队列 |
| 培训机构班主任 | 确认参训学员名单、与市/区/县级政府代表沟通确认意愿与开班安排、选择排期 |
| 县级组织者 | 人工咨询（确认开班意愿）、组织学员完成线上预学习/完成考试、组织学员完成实名身份认证 |
| 意愿用户 | 获取报名链接 → 账号注册 → 报名申请 → 选择培训类型 → 选择参与身份 → 填写基础信息 → 用户选择机构 → 获取机构联系方式 → 在线学习阶段（考试及格）→ 报名成功 → 等候本县有20个完成报名（邀请临近村参与学习）→ 随时查看本区域报名成功人数 |

### 5.3 新型农业经营主体招生流程

与乡村治理骨干班类似，但：
- 意愿用户身份选择不同（市/区/镇政府代表、产业/行业协会代表）
- 开班人数门槛不同（等候本区域、行业报名人数到达60人）
- 当地组织方角色不同

### 5.4 招生流程产品需求

| 序号 | 流程名称 | 业务需求 |
|-----|---------|---------|
| 1 | 招生全流程 | 需要开发组合式的页面功能，用于配制招生任务，让流程各方参与人对整个招生流程有完整的心智认知，提升体验 |
| 2 | 发布招生信息 | 需要可配置的"活动页"功能，用于设计招生推广页面（图文、视频、富文本）；活动页面支持转发朋友圈/个人 |
| 3 | 填写报名申请 | 需要开发报名表单功能，支持富文本填写，多种逻辑选择功能；选择"市/区/县级政府代表"的用户，需要转人工咨询服务，将第三方培训机构用联系方式提供给用户 |
| 4 | 在线学习考试 | 需要开发在线视频学习功能，后台可设置"首次"观看视频无法拖动进度条，并开发"视频倍速播放功能"，最高2倍速，不影响学习；开发在线学习任务之间的步骤锁功能，未完成视频学习播放（进度100%），考试不开启功能；后台可以记录考试后的分数，如多次就统一考试刷题，记录最高分 |
| 5 | 队列管理 | 开发耕耘者全国开班计划与进度大图，以中国地图为背景，展示各个省、市、区的线下开班进度（报名、学习、排期、开班中、已完成），支持分享带报名二维码的朋友圈转发图片；前端展示数据：多少个各地开展培训，多少人参与培训，累计线上学习时长，多少个班正在培训进行中，筹备中；该大图用于多种场景：1.在报名申请、在线学习业务激励学员完成任务，参加培训；2.可放置于线下培训首页，提高耕耘者线下培训的品牌影响力；3.用于农业农村部监督管理线下培训业务；某区域开班人数达标后，自动提醒给对应的培训机构，进行与当地市/县级政府代表沟通开班事宜；正式进入开班队列，需要开发"排期"工具提供给合作机构为班级选定时间，可以选择开班日期，并推送给对接人进行确认 |
| 6 | 线下培训进度管理 | 开展耕耘者全国开班计划与进度数据看班，可支持查询具体某个地区/个人的报名学习情况，同时开发机器人播报功能，每周更新开班数据 |

## 六、开班流程详解

### 6.1 流程概述

开班流程分为三个阶段：**开班准备 → 现场教学 → 评估应用**

**角色与职责**：

| 角色 | 主要职责 |
|-----|---------|
| 监督员（农业农村部、当地组织单位） | 数据监控（申请项目、审批进度）→ 数据监控（查看考勤、课程照片、学生笔记一级满意度打分） |
| 工作组成员 | 审审（3天未审评，发消息催促）→ 发送开班申请成功通知 |
| 机构项目负责人 | 创建开班申请 |
| 班主任 | 设置班级名称、选择开班类型、选择学员类型、导入学员名单、设置开班时间/场地、填写培训目标、设置日程/课程/讲师、设置结格条件/证书 → 提交申请 → 课堂管理工具包（小组积分榜、个人积分榜、学员考勤登记、异常/突发情况上报、每日课堂拍照记录）→ 讲师满意度评价 → 生成结班报告（小组展示照片） |
| 学员 | 收到开班通知 → 每日签到 → 参与学习（每日课程查看、学习笔记、每日培训满意度评价）→ 阶级评价 → 学习记录查看 → 获取奖章（电子培训证明） |
| 讲师 | 签到 → 路演打分 |
| 评委 | 路演打分 |

### 6.2 开班流程产品需求

| 序号 | 流程名称 | 业务需求 |
|-----|---------|---------|
| 1 | 全流程管理 | 需要开发一个从开班申请到培训结束的全流程项目管理产品，可以清楚指引班主任按照产品指引一步步完成开班流程；支持多角色参与开班项目（班主任、监督者、讲师、学员） |
| 2 | 开班申请 | 需要开发之支持多逻辑的开班申请功能，按照指引让班主任完成开班申请；开班申请必须具备审批功能，审批通过后才可以生成班级管理；需要开发学员名单导入功能；审批流需要支持腾讯外部人员参与；完成审批后，需要自动推送开班通知 |
| 3 | 现场教学 | 需要开发签到功能，使用二维码让学员/讲师进行现场签到（近场签到，无法远程扫码）；需要开发班主任课程管理工具包，包括小组/个人积分系统、学员考勤登记、异常/突发情况上报、每日课堂拍照记录；开发词云图功能组件，配合开展教学活动；需要开发多角色之间的满意度评价功能（学员对培训、班主任对老师）；开发路演项目管理功能，支持评委打分、统计；开发电子培训证明体系（发证、查证、展示）；需要开发个人学习档案，沉淀学员在线下培训中的学习记录（作业成果、合照、电子证书、成就）；成就体系需要对完成线下三个学习阶段的的学员 |
| 5 | 学员学习 | 需要开发学员的学习工具包，包括每日的课程列表查看、课件回顾、学习笔记功能（拍照、富文本） |
| 9 | 学员身份认证 | 注册账号新增"耕耘者身份认证"功能，用于识别报名参加培训的用户真实身份，需要开发人工审核功能、配置审批管理员权限功能 |

## 七、资源管理流程

资源管理流程用于指导规范耕耘者线下培训资源，用于保障线下开班工作的顺利开展与提升质量。

**四大资源**：课程、讲师、合作机构、研学基地、教学物资

### 7.1 课程资源管理

**流程**：新增课程 → 使用课程

| 角色 | 新增课程职责 | 使用课程职责 |
|-----|------------|------------|
| 工作组/腾讯管理员 | 查看资源数据 | 查看资源数据 |
| 班主任 | 添加新课程 → 基本信息（名称、封面、学习目标、学习对象、内容简介、作者、授课时长）→ 上传标准课件（PPT/PDF/Word）→ 完成课程入库 | 选择课程 → 课程评价 → 记录使用、归档 |
| 讲师 | - | 下载标准课件 → 上传定制课件（如需）→ 使用课件 → 课程评价 |
| 学员 | - | 课程评价 |

### 7.2 讲师资源管理

**流程**：讲师入驻 → 讲师授课

| 角色 | 讲师入驻职责 | 讲师授课职责 |
|-----|------------|------------|
| 工作组 | 发布讲师招募 → 资格评审 → 讲师入库 | 查看资源数据 |
| 机构负责人 | 转发招募链接 | - |
| 讲师候选人/讲师 | 申请/填写基本信息（姓名、照片、性别、工作单位、工作职位与内容、授课经验、擅长课程、授课经验、推荐单位） | 同意授课 → 授课 → 讲师评价 → 记录使用、归档 |
| 班主任 | - | 选择讲师 → 讲师评价 |
| 学员 | - | 讲师评价 |

### 7.3 合作机构管理

**流程**：机构入驻

| 角色 | 职责 |
|-----|------|
| 工作组/腾讯管理员 | 发布机构招募 → 查看机构数据 → 机构入库 → 设置机构主账号 |
| 申请机构 | 申请/填写基本信息（单位名称、单位属性、营业执照、法人、项目负责人、可提供班级、可交付地点、单位组织规模、推荐单位）→ 设置机构子账号（机构） |

## 九、关键产品能力总结

基于以上需求梳理，耕耘者学习平台的线下培训管理具备以下核心能力：

### 9.1 招生能力
- 招生活动页配置
- 报名表单（多逻辑）
- 在线预学习 + 考试
- 招生队列管理
- 全国开班进度大图
- 排期工具

### 9.2 开班能力
- 开班申请（多逻辑、审批流）
- 学员名单导入
- 班级信息配置（类型、日程、课程、讲师、证书）
- 开班通知推送

### 9.3 现场教学能力
- 二维码签到（近场）
- 课堂管理工具包（积分、考勤、异常上报、拍照记录）
- 词云图教学组件
- 满意度评价（多角色）
- 路演管理 + 评委打分
- 电子培训证明

### 9.4 学员学习能力
- 学习工具包（课程列表、课件回顾、学习笔记）
- 个人学习档案
- 成就体系

### 9.5 资源管理能力
- 课程资源管理（上传、入库、调用、评价）
- 讲师资源管理（招募、入库、选择、评价）
- 合作机构管理（入驻、审核、账号分配）
- 研学基地管理
- 教学物料管理

### 9.6 数据与监控能力
- 招生数据看板
- 开班进度监控
- 培训效果统计
- 机器人播报
