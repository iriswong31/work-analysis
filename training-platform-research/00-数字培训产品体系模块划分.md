# 数字培训产品体系 - 功能模块标准划分

> 更新时间：2026年2月3日
> 
> 本文档基于行业标准和调研，梳理完整的数字培训产品应具备的功能模块体系

---

## 一、划分依据

### 1.1 行业标准参考

**ISO 10015 培训管理体系标准**

ISO 10015是国际标准化组织发布的《质量管理—培训指南》，定义了培训管理的完整流程：

```
培训需求确定 → 培训设计与策划 → 培训实施 → 培训效果评估 → 监视与改进
```

**企业LMS（学习管理系统）通用架构**

业界主流LMS系统（如Canvas、乐享、云学堂等）的功能架构通常包括：
- 用户与组织管理
- 课程内容管理（CMS）
- 学习管理（LMS）
- 知识管理（KM）
- 培训运营管理
- 数据分析与报表

### 1.2 深根者计划特殊需求

基于你提供的材料，深根者计划有以下特殊需求：
- **多层级机构管理**：联合机构 → 地方基金会 → 培训班
- **混合式培训**：线上课程 + 线下走访/研学
- **非强制学习**：以任务引导和证书激励为主
- **标准化培训流程**：签到、纲要、听课、作业、考核

## 二、功能模块体系（建议采用）

基于以上调研，建议按以下 **8大模块** 来盘点各培训平台的功能：

### 模块1：用户与权限管理

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 用户注册/登录 | 支持微信授权、手机号等方式 | 必备 |
| 用户角色定义 | 管理员、讲师、学员等角色区分 | 必备 |
| 权限分配 | 不同角色拥有不同操作权限 | 必备 |
| 用户画像/标签 | 学员背景、岗位、学习偏好等 | 增强 |
| 学习档案 | 个人学习记录、证书、成长轨迹 | 增强 |

**为什么重要**：深根者计划有三类用户（系统管理员、培训管理员、学员），需要清晰的角色权限体系。

---

### 模块2：组织与机构管理

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 多层级组织架构 | 支持总部-分支-部门等层级 | 核心 |
| 机构入驻/审核 | 合作机构的申请与审批 | 核心 |
| 机构分权管理 | 各机构管理自己的培训项目 | 核心 |
| 机构数据隔离 | 各机构数据独立，互不可见 | 增强 |
| 跨机构协作 | 机构间资源共享、联合培训 | 增强 |

**为什么重要**：深根者计划需要"联合机构 → 地方基金会 → 培训班"的三级管理结构，这是核心需求。

---

### 模块3：培训项目管理

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 培训班/项目创建 | 创建培训项目，设置基本信息 | 必备 |
| 学员招募/报名 | 学员报名、审核、分班 | 必备 |
| 培训计划/日程 | 设置培训时间表、阶段安排 | 必备 |
| 培训过程跟踪 | 跟踪项目执行进度 | 必备 |
| 线下活动管理 | 走访、研学等线下活动安排 | 增强 |
| 培训班独立品牌 | 各培训项目有独立入口/小程序 | 增强 |

**为什么重要**：深根者计划有"基础学习→线下走访→考核颁证"三阶段流程，需要完整的项目管理能力。

---

### 模块4：课程内容管理（CMS）

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 视频课程上传/管理 | 录播课程的上传、转码、管理 | 必备 |
| 图文内容管理 | 文章、资料的编辑发布 | 必备 |
| 直播课程 | 实时直播授课 | 增强 |
| 课程分类/标签 | 按主题、难度等分类 | 必备 |
| 课程纲要/目录 | 课程章节结构、学习路径 | 增强 |
| 课程介绍页 | 课程目标、内容简介、证书说明 | 增强 |
| 资源库管理 | 课件、文档等资源统一管理 | 增强 |

**为什么重要**：内容是培训的核心，需要能承载各类课程资源。Coursera的课程介绍、纲要功能是参考点。

---

### 模块5：学习过程管理（LMS）

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 学习进度追踪 | 记录学员学习时长、完成度 | 必备 |
| 签到/打卡 | 线上线下签到功能 | 必备 |
| 学习任务 | 分配学习任务，跟踪完成情况 | 必备 |
| 作业提交 | 学员提交作业，讲师批改 | 增强 |
| 课堂测验 | 学习过程中的随堂测验 | 增强 |
| 学习提醒 | 消息推送提醒学员学习 | 增强 |
| 防溜号机制 | 不定时测验、学习时长验证等 | 增强 |

**为什么重要**：深根者计划需要签到、任务管理功能；Coursera的"防溜号测验"是参考点。

---

### 模块6：考核与认证

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 在线考试 | 组卷、计时考试、自动阅卷 | 增强 |
| 考核规则设置 | 通过条件、补考机制等 | 增强 |
| 证书发放 | 完成培训后颁发电子证书 | 必备 |
| 证书模板管理 | 自定义证书样式 | 增强 |
| 学分管理 | 学分累计、学分兑换 | 增强 |

**为什么重要**：深根者计划以"证书发放"作为激励手段，考核颁证是重要环节（虽然目前还未执行）。

---

### 模块7：互动与社区

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 学员社区/论坛 | 学员交流讨论区 | 增强 |
| 课程评论 | 对课程的评价反馈 | 增强 |
| 问答功能 | 学员提问、讲师解答 | 增强 |
| 学习小组 | 分组学习、小组任务 | 增强 |
| 消息/通知 | 站内信、系统通知 | 必备 |

**为什么重要**：培训不仅是听课，还需要互动交流，提升学习效果。

---

### 模块8：数据统计与运营

| 功能点 | 说明 | 重要性 |
|-------|------|--------|
| 学习数据统计 | 学员数、完成率、学习时长等 | 必备 |
| 培训效果分析 | 考试通过率、满意度等 | 增强 |
| 报表导出 | 数据导出为Excel等格式 | 必备 |
| 数据看板 | 可视化数据展示 | 增强 |
| 运营工具 | 消息推送、活动配置等 | 增强 |

**为什么重要**：培训效果需要数据来衡量，便于持续优化。

## 三、模块重要性分级

根据深根者计划的需求，各模块重要性排序：

| 优先级 | 模块 | 原因 |
|-------|------|------|
| ⭐⭐⭐⭐⭐ | 组织与机构管理 | 深根者核心需求：多层级分权管理 |
| ⭐⭐⭐⭐⭐ | 培训项目管理 | 培训班管理、三阶段流程支持 |
| ⭐⭐⭐⭐ | 学习过程管理 | 签到、任务、进度跟踪 |
| ⭐⭐⭐⭐ | 课程内容管理 | 承载培训课程资源 |
| ⭐⭐⭐ | 用户与权限管理 | 基础能力，各平台都有 |
| ⭐⭐⭐ | 考核与认证 | 证书激励，后续需要 |
| ⭐⭐ | 数据统计与运营 | 运营阶段需要 |
| ⭐⭐ | 互动与社区 | 锦上添花 |

2. **评估标准**：
   - ✅ 完善：功能完整且易用
   - ⚪ 基础：有基础功能但不完善
   - ❌ 无：不具备该功能
   - ❓ 待确认：需要进一步了解

3. **最终产出**：形成功能矩阵对比表，清晰展示各平台在各模块的能力差异
