# 腾讯公益·机构服务平台工具盘点

> **评估目标**：以公益机构全流程视角，盘点机构后台工具
> **评估日期**：2026-01-11

---

## 机构全流程工具盘点

| 一级 | 二级 | 工具介绍 | 使用依赖 | 其他 |
|------|------|----------|----------|------|
| 1. 入驻建档 | 机构信息 | 维护机构主体资料：名称、资质证照、对公账户、法人信息、联系人、机构简介、Logo等 | 机构完成入驻审核；超级管理员权限；变更敏感信息需重新审核 | |
| 1. 入驻建档 | 机构设置 | 账号与权限管理：添加/删除成员、分配角色、设置操作权限、消息通知订阅、安全设置 | 超级管理员权限；成员需通过企业微信/微信认证 | |
| 1. 入驻建档 | 合同中心 | 管理机构与平台签署的各类协议：入驻协议、项目合作协议、资金托管协议等，支持在线签署、查看、下载 | 机构入驻后自动生成基础协议；发起项目/活动时触发补充协议；需管理员或法务角色 | |
| 1. 入驻建档 | 机构运营 | 机构运营能力集合：申请平台运营位/资源位、参与培训/赋能活动、申报行业资助计划、内容/社群运营工具入口 | 机构入驻完成；部分能力需单独申请开通；资源位需排期/审核 | |
| 2. 立项/发起 | 项目管理 | 项目全生命周期管理：创建项目、编辑项目信息、管理项目状态（草稿/审核中/上线/暂停/结项）、配置多端发布 | 机构认证通过；项目材料齐备；非公募需公募机构认领；通过平台审核 | |
| 2. 立项/发起 | 审核管理 | 机构侧审核工作台：查看/处理待审核事项（项目上线、信息变更、专项基金、帮扶对象、一起捐、透明度披露等） | 机构已入驻；具备对应审核角色权限；存在待审核对象 | |
| 3. 拓流/动员 | 一起捐管理 | 社交化组队筹款工具：创建一起捐活动、管理参与小队/个人、查看筹款进度、排行榜、传播数据 | 已有上线项目；项目支持一起捐玩法；依赖微信社交传播 | |
| 3. 拓流/动员 | 小红花码 | 线下场景公益参与入口：生成带项目/活动信息的二维码，用户扫码可参与捐赠/领取小红花，支持统计扫码数据 | 已有上线项目/活动；配置扫码落地页；线下物料制作与投放 | |
| 3. 拓流/动员 | 海报工具 | 项目传播物料生成器：选择模板、自动填充项目信息、生成可分享海报图片，用于朋友圈/社群传播 | 已有上线项目；项目封面/信息完整；依赖微信传播场景 | |
| 3. 拓流/动员 | 分分捐 | 微信支付场景小额捐赠工具：用户支付后可选择"凑整捐"，形成持续小额捐赠流 | 机构/项目被纳入分分捐承接池；依赖微信支付场景入口；平台统一配置 | |
| 3. 拓流/动员 | 群发短信 | 批量触达捐赠人/参与者：选择发送对象、选择/编辑短信模板、发送运营通知/进展同步/活动召回，查看发送统计 | 有捐赠人/参与者数据；用户授权接收短信；短信配额；模板需审核 | |
| 4. 执行/履约 | 参与者管理 | 项目参与者数据管理：查看/导出捐赠人列表、志愿者列表、一起捐小队成员、活动报名者等，支持筛选/分组/标签 | 项目/活动产生参与数据；部分数据需用户授权；导出需权限 | |
| 4. 执行/履约 | 捐赠人服务 | 捐赠人关系维护工具：捐赠人档案、分层管理（大额/月捐/普通）、爱心回礼配置、感谢信/证书发送、专属沟通 | 有捐赠人数据；部分功能需用户授权；回礼需物流/成本 | |
| 4. 执行/履约 | 爱心券 | 数字化救助券工具：创建券活动、录入受助人名单、录入合作商家、发放券、受助人刷脸领取、商家扫码核销、资金自动结算 | 有资金/资源来源；录入受助人信息；签约合作商家；配置核销规则 | |
| 4. 执行/履约 | 活动报名 | 线上/线下活动报名管理：创建报名活动、收集报名信息、审核报名者、发送通知、现场签到、导出名单 | 创建活动；配置报名表单；可选：短信/微信通知、签到码 | |
| 4. 执行/履约 | 数字化执行工具 | 项目执行过程数字化工具集：执行日志记录、受益人签收确认、物资发放登记、现场照片/定位采集、执行进度跟踪 | 项目进入执行期；执行人员使用小程序/App；可能需要定位/拍照权限 | |
| 4. 执行/履约 | 数字工具箱 | 公益机构数字化能力超市：展示平台提供的各类数字工具（电子签章、云存储、协作文档等），机构可申请开通/领取权益 | 机构入驻；按工具单独申请；部分工具有配额/有效期 | |
| 4. 执行/履约 | 更多功能 | 新能力/实验功能聚合入口：承载平台灰度测试的新工具、合作伙伴提供的外部能力、临时性活动工具等 | 视具体功能而定 | |
| 5. 透明/披露 | 项目透明度 | 项目信息披露管理：按平台规则定期提交项目进展、资金使用明细、受益人故事、结项报告等，披露内容公开展示在项目页 | 项目上线；按规则定期提交；披露内容需审核；影响项目信任评级/活动准入 | |
| 6. 对账/结算 | 活动对账 | 资金对账与结算管理：查看活动/项目资金流水、核对账单明细、申请结算/提现、下载对账单/流水明细 | 产生资金流水；配置结算账户；财务角色权限；满足结算条件 | |
| 6. 对账/结算 | 捐赠票据配置 | 捐赠发票/收据服务配置：设置机构开票信息、票据类型、开票规则、票据说明文案、处理联系方式 | 机构具备开票资质；配置开票信息；捐赠人发起申请后处理 | |
| 6. 对账/结算 | 导出任务列表 | 数据导出任务管理：发起数据导出、查看导出进度、下载导出文件、管理历史导出记录 | 有可导出数据；具备导出权限；大数据量走异步任务 | |
| 7. 复盘/沉淀 | 机构数据 | 机构维度数据看板：机构整体筹款金额/笔数、项目数量/状态分布、捐赠人数/复捐率、传播数据等 | 机构有业务数据；数据查看权限 | |
| 7. 复盘/沉淀 | 项目数据 | 项目维度数据看板：单项目筹款进度、捐赠人分布、传播渠道效果、执行进度、披露完成度等 | 项目上线并产生数据 | |
| 7. 复盘/沉淀 | 产品数据 | 工具/产品使用数据：各工具的使用次数、效果转化、功能渗透率等 | 使用过对应工具 | |
| 7. 复盘/沉淀 | 活动数据 | 活动维度数据看板：参与的平台活动（如99公益日）数据、机构自办活动数据、一起捐活动数据等 | 参与/发起活动 | |
| 7. 复盘/沉淀 | 全年行业资助 | 行业资助计划数据：查看平台年度资助计划、机构申报记录、资助结果、资助金额使用情况等 | 机构参与资助计划；按规则申报 | |
